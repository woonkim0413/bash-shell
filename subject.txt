쉘이 반드시 구현해야 할 기능

- 프롬프트 표시
	새 명령을 기다릴 때마다 사용자에게 프롬프트를 출력한다.

- 히스토리 기능
	이전에 입력한 명령들을 저장하고 재사용할 수 있어야 한다.

- 명령 검색 및 실행
	PATH 환경변수 혹은 상대경로·절대경로를 활용해 올바른 실행파일을 찾아 실행한다.

- 전역 변수 제한
	받은 시그널을 나타내기 위한 전역 변수는 하나만 사용해야 하며, 
	여기에 쉘의 주요 데이터구조를 직접 노출해서는 안 된다.

- 인용부호 및 특수문자 처리
	닫히지 않은 인용부호나 \, ; 등 주제에서 요구하지 않는 특수문자는 해석하지 않는다.

- 싱글 쿼트 처리 ('…')
	작은따옴표 안에서는 메타문자를 전혀 해석하지 않는다.

- 더블 쿼트 처리 ("…")
	큰따옴표 안에서는 $만 확장하고, 나머지 메타문자는 해석하지 않는다.

- 리디렉션 구현
	< : 입력 리디렉션
	> : 출력 리디렉션 (덮어쓰기)
	<< : here‑document (구분자 입력 전까지 표준 입력으로 처리; 히스토리에는 남기지 않아도 됨)
	>> : 출력 리디렉션 (추가 모드)

- 파이프 구현 (|)
	파이프라인을 구성하여 앞 명령의 출력을 뒤 명령의 입력으로 연결한다.

- 환경 변수 확장 ($VAR)
	$ 뒤에 오는 문자열을 환경변수 값으로 치환한다.

- 종료 상태 확장 ($?)
	마지막으로 실행된 포어그라운드 파이프라인의 종료 상태 값을 치환한다.

- 시그널 처리
	Ctrl‑C: 새 줄에 프롬프트를 다시 출력
	Ctrl‑D: 쉘 종료
	Ctrl‑\: 아무 동작도 하지 않음

- 내장 빌트인(builtin) 구현
	echo (옵션 -n 지원)
	cd (상대/절대 경로만)
	pwd (옵션 없음)
	export (옵션 없음)
	unset (옵션 없음)
	env (옵션 및 인자 없음)
	exit (옵션 없음)

메모리 관리

readline() 자체의 누수는 예외이나, 
사용자가 작성한 모든 코드는 메모리 누수 없이 해제되어야 한다.
